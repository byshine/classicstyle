Side note: We can also choose the direction of the flex layout with flex-direction: row, flex-direction: column, flex-direction: row-reverse, flex-direction: column-reverse


Now the interesting part is the controlling the actual flex items size & proportion. This is done by adding flex. Flex takes three values, with them being flex-grow, flex-shrink, flex-basis.

Flex-grow works as a unitless proportion. If we set flex: 1 to all the 3 divs, then each will take up equal sizes.

Blue
Red
Green

Notice how flex: 1 provided nearly identical/better results than setting each of the div's width to 33.3333%

The second part of flex, flex-shrink will be covered later.

Flex-basis is the minimum size value that the flex item should occupy. This is not unitless, so you can give it a percentage like we did, 33.3333%. When used in combination with flex-grow, flex basis will give the minimum space, and rest of the space exceeding that will be proportional. This may or may not be what you want. Notice when compared with the traditional grid systems using floats, flexbox still is really the same. Flexbox was not made for grids, but for layouts. However, one can emulate a grid system with flexbox just as it has been done with floats. But the nice thing about flexboxes are that it comes with variety of positioning concepts that we normally use. For example, we can use align-items for vertical centering, and justify-content for horizontal centering.
