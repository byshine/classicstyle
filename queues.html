Queues

Queues in essence are linked lists with a fixed mechanism of removing and adding nodes. So if you already know how linked lists work, you already know how to queues work. Think of a stack implemented with linked lists. We push nodes and pop the last one, giving us a LIFO (Last In First Out) structure. Queues are FIFO (First in First Out) structures. All we need to do is add nodes from one side and remove from the other. Now Priority Queues are different beasts. Let's not get into that for now. We will be modifying our singly linked list implementation for this queue. 

function Node( data ) {
    this.data = data;
    this.next = null;
}

function Queue() {
    this.head = null;
    this.tail = null;
    this.size = 0;
}
Adding nodes in a queue is exactly the same as adding a node in a singly linked list. But instead of add, we'll call this enqueue 
Queue.prototype.enqueue = function( data ) {
    let node = new Node( data );

    if ( !this.head ) {
        this.head = node;
        this.tail = node;
        this.size++;
        return node;
    }

    this.tail.next = node;
    this.tail = node;
    this.size++;
    return node;
}
For popping, or usually called dequeue, we always remove the head. This makes dequeue very simple to implement. It doesn't make sense to remove any nodes other than the head. (In the real world, you may have inserted something in queue by accident, and may want to remove such item so yes, it actually does make sense to remove a node besides the head- but we have already seen how to remove nodes in a list. The same method can apply here and that method is not called dequeue, it should be called remove.)
